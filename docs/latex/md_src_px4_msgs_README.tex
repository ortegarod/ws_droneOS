\doxysubsection*{Messages Sync from PX4}

When PX4 message definitions in the {\ttfamily main} branch of \href{https://github.com/PX4/PX4-Autopilot}{\texttt{ PX4 Autopilot}} change, a \href{https://github.com/PX4/PX4-Autopilot/blob/main/.github/workflows/metadata.yml\#L119}{\texttt{ CI/\+CD pipeline}} automatically copies and pushes updated ROS message definitions to this repository. This ensures that this repository {\ttfamily main} branch and the PX4-\/\+Autopilot {\ttfamily main} branch are always up to date. However, if you are using a custom PX4 version and you modified existing messages or created new one, then you have to manually synchronize them in this repository\+: \doxysubsection*{Manual Message Sync}


\begin{DoxyItemize}
\item Checkout the correct branch associated to the PX4 version from which you detached you custom version.
\item Delete all {\ttfamily $\ast$.msg} and {\ttfamily $\ast$.srv} files in {\ttfamily msg/} and {\ttfamily srv/}.
\item Copy all {\ttfamily $\ast$.msg} and {\ttfamily $\ast$.srv} files from {\ttfamily PX4-\/\+Autopilot/msg/} and {\ttfamily PX4-\/\+Autopilot/srv/} in {\ttfamily msg/} and {\ttfamily srv/}, respectively. Assuming that this repository and the PX4-\/\+Autopilot repository are placed in your home folder, you can run\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{rm -\/f \string~/px4\_msgs/msg/*.msg}
\DoxyCodeLine{rm -\/f \string~/px4\_msgs/srv/*.srv}
\DoxyCodeLine{cp \string~/PX4-\/Autopilot/msg/*.msg \string~/px4\_msgs/msg/}
\DoxyCodeLine{cp \string~/PX4-\/Autopilot/msg/versioned/*.msg \string~/px4\_msgs/msg/}
\DoxyCodeLine{cp \string~/PX4-\/Autopilot/srv/*.srv \string~/px4\_msgs/srv/}

\end{DoxyCode}
 {\bfseries{Note\+:}} The ROS 2 message generation pipeline requires all messages to be directly under {\ttfamily msg/} and doesn\textquotesingle{}t support sub-\/directories.
\end{DoxyItemize}\hypertarget{md_src_px4_msgs_README_autotoc_md10}{}\doxysection{Install, build and usage}\label{md_src_px4_msgs_README_autotoc_md10}
